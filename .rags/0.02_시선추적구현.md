- # EyeTracker 개발 가이드: 0.01 ~ 0.02 단계 구현
- 본 문서는 구현계획서의 초기 단계를 실제 코드로 구현한 명세입니다.

- 대주제: 눈 트래킹 기술의 실제 구현 가능성 및 원리
    - 소주제: 플랫폼별 기술 스택 분석
        - 세부사항: iOS (ARKit): FaceID 센서를 통해 각 눈의 변환(Transform) 및 시선 지점(LookAtPoint)을 하드웨어 레벨에서 제공함 (정밀도 매우 높음)
        - 세부사항: Android (ARCore): Augmented Faces API를 통해 얼굴 메시 데이터를 추출하고, 동공의 위치 변화를 기하학적으로 계산하여 시선 벡터 도출 가능
        - 세부사항: 통합 플랫폼 (AR Foundation): 유니티 AR Foundation의 ARFaceManager를 사용하여 양대 플랫폼의 데이터를 추상화된 인터페이스로 획득 가능
    - 소주제: 실시간 시선 추적의 한계 및 보정법
        - 세부사항: 노이즈 (Noise): 모바일 카메라의 초당 프레임 제한으로 인한 떨림은 칼만 필터(Kalman Filter)로 해결 가능
        - 세부사항: 하드웨어 의존성: 전면 TrueDepth 카메라 유무에 따라 정밀도 차이가 발생할 수 있음

- 대주제: [0.02 단계] 유니티 시선 추적 핵심 스크립트 (C#)
    - 소주제: ARFaceManager를 이용한 시선 데이터 추출 소스코드
    - 세부사항: 아래 코드는 유니티 AR Foundation에서 눈의 위치와 방향 정보를 동기화하는 핵심 로직입니다.

```csharp
using UnityEngine;
using UnityEngine.XR.ARFoundation;
using UnityEngine.XR.ARSubsystems;

public class EyeGazeProvider : MonoBehaviour
{
    private ARFaceManager faceManager;
    private Vector3 leftEyePos;
    private Vector3 rightEyePos;
    private Vector3 gazeDirection;

    void Awake()
    {
        faceManager = GetComponent<ARFaceManager>();
    }

    void OnEnable()
    {
        faceManager.facesChanged += OnFacesChanged;
    }

    void OnDisable()
    {
        faceManager.facesChanged -= OnFacesChanged;
    }

    private void OnFacesChanged(ARFacesChangedEventArgs args)
    {
        foreach (var face in args.updated)
        {
            if (face.trackingState == TrackingState.Tracking)
            {
                // 왼쪽/오른쪽 눈의 위치 및 회전값 추출
                leftEyePos = face.leftEye.position;
                rightEyePos = face.rightEye.position;

                // 시선 방향 계산 (양안의 중간 지점에서 정면 벡터 추출)
                gazeDirection = (face.leftEye.forward + face.rightEye.forward).normalized;
                
                // 시뮬레이션 타겟(열기구)과의 충돌 여부 확인을 위한 레이캐스트 준비 완료
                ProcessGaze(leftEyePos, rightEyePos, gazeDirection);
            }
        }
    }

    private void ProcessGaze(Vector3 left, Vector3 right, Vector3 direction)
    {
        // 다음 단계(0.03): 칼만 필터를 통한 시선 안정화 로직이 여기에 위치함
        Debug.Log($"Gaze Vector: {direction}");
    }
}
```

- 대주제: 구현 시 주의사항 및 세부 정보
    - 소주제: 성능 최적화
        - 세부사항: 매 프레임 발생하는 Face Mesh 업데이트는 가비지 컬렉션을 유발하므로 캐싱 필수
        - 세부사항: 60fps 유지를 위해 시선 연산은 비동기 또는 Job System 활용 고려
